\chapter{Introduction}
\label{cap:introduction}

\chapterquote{Document the code.}{Pedro Pablo}

This report describes the study, design, and implementation of a technological solution that enables the use of \textbf{augmented reality (AR)} through visual marker detection in live broadcasts, based on the open-source platform \textbf{OBS Studio}. The project is situated in a context where the audiovisual world is evolving rapidly, combining real-time content creation with immersive technologies that are transforming the way digital content is communicated and consumed.

The starting point of this work lies in how AR has gone from being an experimental curiosity to becoming a standard tool in audiovisual production. Today, it is no longer surprising to see how news programmes, sports broadcasts, or entertainment shows include 3D graphics, animations, and interactive elements overlaid on real environments, creating a much more visual and engaging experience for viewers.

With this project, we aim to explore to what extent it is possible to bring these ideas into a specific domain: the \textbf{live \textit{streaming} of programming contests}. Although AR is still not widely used in this type of content, this technology has great potential to make broadcasts clearer, more dynamic, and more engaging, while also contributing to raising the profile of these technical competitions.


\section{Motivation}
\label{sec:motivacion_en}

In today's professional television broadcasts, particularly live news programmes, \textbf{augmented reality (AR)} is used as a resource to reinforce and clarify information. This trend demonstrates the potential of AR as a visual communication tool in real time, providing clarity, dynamism, and greater appeal to content.

In the domain of live \textit{streaming}, one of the most widely used tools is \textbf{OBS Studio}, owing to its free and open-source nature and the large community that supports it. However, the platform does not offer native support for integrating advanced AR effects, which limits its use in contexts where such capabilities would be desirable.

A particularly representative domain in which to explore this possibility is that of \textbf{university programming contests}. These events generate very rich real-time information: rankings updated instant by instant, submission verdicts, penalty times, and, in the final moments, the dramatic resolution of a frozen scoreboard. Yet the broadcasts of these contests are typically simple, with a fixed camera over the participants' desks and little visual information for the viewer. AR offers a concrete opportunity here: overlaying that structured information directly onto the live image, bringing the experience of these university competitions closer to the visual standards of professional sports broadcasts.

The Faculty of Computer Science of the Complutense University of Madrid (FDI-UCM) organises programming contests in the ICPC style every year, bringing together teams of undergraduate students. The live broadcast of these competitions, typically through \textit{streaming} platforms, represents an ideal scenario in which to explore the use of AR: there is real-time structured information, a technical audience that appreciates precise visual representations, and a simple audiovisual production that could benefit greatly from a layer of augmented content. This specific context serves as the main use case and guides many of the design decisions of the system.

The motivation for this project stems from the combination of these observations: to explore the possibility of equipping OBS Studio with AR capabilities and applying them practically to the broadcasting of programming contests. The aim is to lay a technological foundation that can open new opportunities for innovation in the creation of interactive, real-time digital content, demonstrable in a concrete technical use case.


\section{Objectives}
\label{sec:objetivos_en}

The main objective of this project is to incorporate \textbf{augmented reality (AR)} functionalities in real time into \textbf{OBS Studio}. To this end, a native OBS Studio filter is developed that can integrate synthetic content directly into the video stream of a broadcast, without additional latency from external processes and without modifying the software core.

This approach entails resolving a set of technical challenges that are addressed as concrete objectives of the project:

\begin{itemize}
  \item Detect \textbf{ArUco markers} in each camera frame and estimate the marker pose in real time using OpenCV's PnP algorithm, in order to determine the position and orientation of the virtual content to be overlaid.
  \item Implement the \textbf{coordinate transformation chain} between the OpenCV and OBS coordinate spaces, resolving the axis convention change that introduces a $180^\circ$ rotation about the horizontal axis.
  \item Develop the four \textbf{operating modes} of the plugin: superimposition of an arbitrary 3D model anchored to the marker, an analogue countdown clock synchronised with the contest time, a real-time contest \textit{scoreboard}, and individual team information for the team detected on camera.
  \item Integrate connectivity with the automatic judging platform \textbf{DOMjudge} through its REST API, using \textbf{libcurl} as the HTTP library, to periodically retrieve rankings, verdicts, and contest times from within the plugin.
  \item Design a \textbf{two-thread architecture} that separates network requests from the rendering cycle, so that network latency never blocks \textit{frame} production.
  \item Develop an independent \textbf{camera calibration tool} that generates the intrinsic parameters file and distortion coefficients loaded by the plugin at startup.
  \item Apply a \textbf{smoothing filter} over the pose estimate to eliminate the visual jitter caused by camera noise and video compression.
  \item Provide a \textbf{user interface} integrated into OBS that allows the operator to configure and change plugin parameters live, without restarting the scene.
\end{itemize}

As a result, the system demonstrates the feasibility of adding a complete data-driven AR \textit{pipeline} to OBS Studio while maintaining the native filter architecture, and opens the door to its adaptation to other types of live broadcast beyond programming contests.

The first phase will be devoted to the state-of-the-art study: a review of AR systems in live broadcasts, a comparative evaluation of available SDKs, and an analysis of the OBS Studio plugin architecture. The results of this research and the technological decisions taken will be presented in the chapter on the state of the art.

The second phase will aim to establish the system architecture and verify that the chosen approach is technically feasible. The plugin structure will be designed and an initial prototype will be built to validate the basic superimposition of 3D content over the camera signal.

The third phase will be the most extensive and will include the complete development of the plugin. The technical description chapters document in detail all the work carried out during this stage.

The fourth phase will be devoted to system stability testing, code review, and the writing of this report.

The rest of the report is organised as follows. The state-of-the-art chapter analyses AR technology: its types, main applications, available SDKs, the justification of the technological choice, the OBS Studio architecture, and its plugin system. The programming contests chapter introduces the domain of programming contests and the DOMjudge platform, which acts as the data source for the dynamic part of the system. The AR description chapter describes the design and implementation of the augmented reality filter in OBS: marker detection, pose estimation, 3D rendering, and AR mode configuration. The programming contests description chapter explains the extension of the filter for programming contests, including dynamic elements, DOMjudge connectivity, and visualisation modes.
